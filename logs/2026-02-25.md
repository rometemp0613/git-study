## 학습 내용

### tag - 버전 표시
- lightweight tag vs annotated tag (실무에선 annotated)
- tag는 별도로 push 필요 (git push origin --tags)
- tag 생성, 조회, 삭제, 원격 push/삭제
- git tag -l "패턴" 필터링 (-l 필수)

### 시맨틱 버전 관리
- MAJOR.MINOR.PATCH 구조
- 상위 올리면 하위 리셋 규칙
- pre-release: alpha → beta → rc → 정식
- v0.x.x = 초기 개발, v1.0.0 = 안정판 선언

## 소감
- tag와 시맨틱 버전을 함께 배우니 자연스럽게 연결됨
- 두 주제 모두 비교적 직관적이라 빠르게 진행

# 2026-02-22 학습 일지 (2)

## 주제: Fork

### 배운 내용
- Fork: GitHub → GitHub 서버 간 레포 복사 (내 계정에)
- Fork vs Clone: 서버 간 복사 vs 서버→로컬 복사
- Remote 2개 구조: origin(내 Fork) + upstream(원본)
- 오픈소스 기여 흐름: Fork → Clone → 브랜치 → 작업 → push → PR
- Upstream 동기화: fetch upstream → merge → push origin

### 실습 내용
- `gh repo fork octocat/Hello-World --clone=true`로 Fork + Clone + upstream 한 번에
- `git remote -v`로 origin/upstream 확인
- 브랜치 생성 → 파일 수정 → 커밋 → push 실습
- `git fetch upstream` → `git merge upstream/master` 동기화 실습
- 연습 후 로컬 삭제 + `gh repo delete`로 Fork 삭제

### 퀴즈 결과
- origin/upstream 역할 설명 부족 → 정리 완료
- `gh repo fork --clone=true`의 3가지: Fork + Clone + upstream 설정
- upstream 동기화 안 하면 PR 충돌 가능 → 정답

### 학습한 주제
- git diff - 변경사항 비교
- git reset - 되돌리기 (히스토리 변경)
- git revert - 안전하게 되돌리기 (새 커밋 생성)
- .gitignore - 추적 제외 파일 설정

### 새로 배운 명령어

| 명령어 | 설명 |
|--------|------|
| `git diff` | Working Directory ↔ Staging Area 비교 |
| `git diff --staged` | Staging Area ↔ 최신 커밋 비교 |
| `git diff HEAD` | Working Directory ↔ 최신 커밋 비교 |
| `git reset HEAD 파일` | unstage (staging → working) |
| `git reset --soft HEAD~1` | 커밋 취소, 변경사항 staged 유지 |
| `git reset --hard HEAD~1` | 모두 삭제 |
| `git revert HEAD` | 최신 커밋 취소 (새 커밋 생성) |

### 메모 / 느낀 점
- reset의 3가지 모드: `--soft`, `--mixed`(기본), `--hard`
- **push한 커밋은 reset 금지, revert 사용!**
- reset은 히스토리 변경, revert는 히스토리 유지
- .gitignore는 커밋해야 팀원과 공유됨
- 이미 추적 중인 파일은 `git rm --cached` 후 .gitignore에 추가

## 주제: HEAD 개념

### 배운 내용

1. **HEAD란**: 현재 내가 서 있는 위치를 가리키는 포인터
2. **HEAD~n**: n세대 위 조상 (첫 번째 부모만 따라감)
3. **HEAD^**: 부모 커밋 (숫자 없으면 = HEAD~1)
4. **HEAD^2**: 머지 커밋의 두 번째 부모
5. **~와 ^ 차이**: ~는 몇 세대, ^뒤 숫자는 몇 번째 부모
6. **.git/HEAD**: HEAD가 뭘 가리키는지 저장된 실제 파일

### 실습

- `git diff HEAD~1` - 직전 커밋과 비교
- `git show HEAD~2 --oneline --no-patch` - 이전 커밋 정보 확인
- `cat .git/HEAD` - HEAD 내부 파일 확인
- `git diff HEAD~3 --name-only` - 변경된 파일 목록
- `git log --oneline HEAD~3..HEAD` - 범위 지정 로그

## 주제: Detached HEAD + checkout vs switch/restore

### 배운 내용

1. **Detached HEAD**: HEAD가 브랜치 없이 커밋을 직접 가리키는 상태
2. **진입 방법**: `git checkout <해시>` 또는 `git switch --detach <해시>`
3. **미아 커밋**: detached 상태에서 커밋 후 브랜치 안 만들고 떠나면 발생
4. **미아 구출**: `git branch <이름> <해시>`로 브랜치 연결
5. **checkout이 분리된 이유**: 브랜치 전환 + 파일 복원을 하나가 담당해서 혼란
6. **switch**: 브랜치 전용 명령어
7. **restore**: 파일 복원 전용 (스테이징 취소, 변경 취소, 특정 커밋에서 복원)

### 실습

- `git checkout <해시>`로 detached HEAD 진입
- `cat .git/HEAD`로 해시가 직접 적힌 것 확인
- `ls`로 과거 시점 파일 확인
- detached 상태에서 커밋 후 브랜치 없이 떠나기 → 미아 커밋 체험
- `git branch rescue <해시>`로 미아 커밋 구출

### 퀴즈 결과

- 3/3 핵심 이해 (세부 표현 일부 수정)
- `git switch --detach`의 정확한 옵션명 학습

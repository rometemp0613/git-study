## 오늘 배운 것

### remote, push, pull, fetch, clone - 원격 저장소 기초

원격 저장소와 로컬 저장소를 연결하고 동기화하는 핵심 명령어들.

### 핵심 명령어

| 명령어 | 용도 |
|--------|------|
| `git clone <URL>` | 원격 저장소를 로컬에 복제. 자동으로 origin 설정 |
| `git remote -v` | 연결된 원격 저장소 목록과 URL 확인 |
| `git push origin main` | 로컬 커밋을 원격에 업로드 |
| `git push -u origin main` | upstream 설정 + push (이후 `git push`만으로 가능) |
| `git fetch` | 원격 변경사항을 원격 추적 브랜치에만 다운로드 |
| `git pull` | fetch + merge. 원격 변경사항을 가져와서 바로 병합 |

### fetch vs pull

| 구분 | fetch | pull |
|------|-------|------|
| 동작 | 다운로드만 | 다운로드 + 병합 |
| 안전성 | 안전 (확인 후 병합 가능) | 바로 병합 (충돌 가능) |
| 업데이트 대상 | 원격 추적 브랜치만 | 원격 추적 브랜치 + 로컬 브랜치 |

### 원격 추적 브랜치

- **origin/main**: 마지막으로 확인한 원격 저장소의 상태를 기억하는 읽기 전용 포인터
- `git branch -r`: 원격 추적 브랜치만 보기
- `git branch -a`: 로컬 + 원격 추적 브랜치 모두 보기
- `git branch -vv`: upstream 연결 상태와 ahead/behind 확인
- fetch, push, pull 시 업데이트됨

### 핵심 포인트

- fetch는 스테이징이 아니라 **원격 추적 브랜치**를 업데이트
- push도 origin/main을 업데이트함
- fetch 후 `git diff main origin/main`으로 차이를 확인하고 merge하는 게 안전

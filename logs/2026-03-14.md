## 주제
7단계: Git 내부 구조 — Git 객체 (blob, tree, commit)

## 학습 내용

### Git 객체 3가지
- **blob**: 파일 내용만 저장 (이름 모름)
- **tree**: 폴더 구조 저장 (이름 → blob/tree 매핑)
- **commit**: tree + parent + author + message

### 핵심 원리
- Git은 키-값 저장소 (SHA-1 해시 = 키)
- 같은 내용은 한 번만 저장 (blob 재사용)
- 파일 이름 변경 시 blob 재사용, tree만 변경

### 실습
- `git cat-file -p HEAD`로 커밋 객체 직접 확인
- 커밋 → tree → blob 체인을 따라가며 내부 구조 탐색
- `git cat-file -t`로 객체 타입 확인

### 명령어
- `git cat-file -p <해시>`: 객체 내용 출력
- `git cat-file -t <해시>`: 객체 타입 확인

## 오늘 배운 것

### Rebase
- 브랜치를 다른 브랜치 위로 "옮겨 심는" 기능
- merge와 달리 히스토리가 일직선으로 깔끔해짐
- 커밋 해시가 바뀌기 때문에 공유된 커밋에는 사용 금지

### Interactive Rebase (git rebase -i)
커밋 히스토리를 세밀하게 편집하는 기능

| 명령어 | 설명 |
|--------|------|
| pick | 커밋 그대로 사용 |
| squash | 이전 커밋과 합침 |
| reword | 커밋 메시지만 수정 |
| drop | 커밋 삭제 |

### 충돌 해결
- rebase 중 충돌 발생 시 파일에서 충돌 마커(<<<, ===, >>>)를 수정
- `git add <파일>` 후 `git rebase --continue`

## 실습 내용
1. 기본 rebase - feature 브랜치를 main 위로 옮기기
2. squash - 지저분한 커밋 3개를 1개로 합치기
3. 충돌 해결 - 같은 파일 수정 후 rebase
4. reword - 오타 있는 커밋 메시지 수정 ("결게" → "결제")
5. drop - 테스트용 커밋 삭제

## 핵심 포인트
> **"이미 공유한 커밋은 rebase하지 마라"**

특히 main 브랜치에서 rebase하면 위험!

## Vim 단축키 (git rebase -i 시 유용)
- `ESC` → Normal 모드
- `i` → Insert 모드
- `cw` → 단어 지우고 입력
- `:wq` → 저장 후 종료

## 오늘 배운 내용

### GitHub Flow (5단계: 협업 전략)

- **GitHub Flow 개념**: main + feature 브랜치만 쓰는 간소화된 전략
- **6가지 규칙**: main 항상 배포 가능, main에서 브랜치, 설명적 이름, 수시 push, PR 필수, 머지 즉시 배포
- **Git Flow와 비교**: develop/release/hotfix 없음, 훨씬 단순
- **적합한 환경**: 웹앱, SaaS, 수시 배포, 소규모 팀
- **핫픽스 처리**: 별도 규칙 없이 feature 브랜치와 동일하게 처리

## 실습

- GitHub Flow 전체 흐름을 직접 실행 (이미 매번 해왔던 방식!)
  - main pull → 브랜치 생성 → 커밋 → push → PR → merge → 정리

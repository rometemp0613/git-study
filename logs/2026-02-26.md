## 학습 주제
- reflog - 실수로 삭제한 커밋 복구

## 배운 내용
- reflog(Reference Log): HEAD 이동 기록을 저장하는 로컬 전용 로그
- git log vs git reflog 차이: log는 커밋 히스토리, reflog는 행동 기록
- reflog 특징: 로컬 전용, 90일 보관, clone하면 없음
- reflog 출력 읽는 법: 해시 / HEAD@{n} / 행동 설명
- 복구 방법: reset --hard(커밋 복구), branch(브랜치 복구)
- reset vs branch 차이: reset은 현재 브랜치 이동, branch는 새 브랜치 생성

## 실습
- 연습용 저장소에서 커밋 생성 → reset --hard로 삭제 → reflog로 복구
- 브랜치 삭제 후 reflog에서 해시 찾아서 브랜치 복구
- rebase squash 후 reflog로 원래 상태 복구

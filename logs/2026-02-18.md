## 복습 퀴즈

### 2-practicing 복습 (5문제)
- ⭕ cherry-pick 기본 → 3-mastered 승급
- ⭕ squash/reword/edit → 3-mastered 승급
- ❌ restore 위험성 (핵심: 영구 삭제)
- △ cherry-pick --no-commit 용도 (핵심: 여러 커밋을 하나로 합칠 때)
- △ git diff --staged (add 후 변경사항 보기)

### 1-new 복습 (6문제)
- ⭕ upstream (-u) → 2-practicing 승급
- ⭕ fetch→확인→merge 흐름 → 2-practicing 승급
- ⭕ gh pr create → 2-practicing 승급
- ❌ git fetch (기억 못함)
- △ origin/main 업데이트 시점 (merge로는 안 됨)
- ❌ PR 머지 3가지 방식 커밋 수 (Merge=N+1, Squash=1, Rebase=N)

## 새로 배운 것: 브랜치 보호 규칙

### 핵심 개념
- GitHub Ruleset으로 브랜치 보호 규칙 설정
- Require PR: main 직접 push 차단
- Block force pushes: --force도 차단
- Restrict deletions: 브랜치 삭제 금지

### 실습 내용
1. main 직접 push → 성공 (보호 규칙 없을 때)
2. Ruleset 설정 (main-protection)
3. main 직접 push → 거부됨! (보호 규칙 작동 확인)
4. force push → 거부됨!
5. feature 브랜치 → PR → 머지 워크플로우 연습
6. git fetch --prune으로 삭제된 원격 브랜치 정리

### 실수에서 배운 것
- 커밋 오타(`commmit`)로 커밋 실패 → 커밋 안 된 채 push → 잘못된 커밋이 PR에 포함됨
- 브랜치 만들 때 main에 불필요한 커밋이 있으면 그게 딸려감
- divergent branches: 로컬/원격이 다른 커밋을 가지고 있을 때 발생

## 주제
7단계: Git 내부 구조 — 해시 함수 (SHA-1, 커밋 ID의 원리)

## 학습 내용

### 해시 함수 핵심 특성 4가지
- **고정 길이 출력**: 입력 크기 무관하게 항상 같은 길이
- **단방향**: 해시값으로 원본 역추적 불가
- **결정적**: 같은 입력 → 항상 같은 해시
- **눈사태 효과**: 1글자만 바꿔도 해시 완전히 달라짐

### SHA-1
- 160비트 (40자리 16진수) 출력
- Git의 객체 식별자로 사용
- 2017년 Google이 충돌 공격 성공 (SHAttered)
- SHA-256으로 전환 중 (실험적 지원)

### Git의 해시 계산 공식
- `SHA-1("{타입} {크기}\0{내용}")`
- blob: `blob 5\0hello` → sha1sum으로 직접 검증 성공

### 해시 체인
- 커밋 안에 parent 해시가 포함됨
- 과거 커밋 수정 → 해시 변경 → 연쇄 반응 → 조작 즉시 발각
- root commit은 parent가 없음 (체인의 시작점)

## 실습
- `git hash-object --stdin`으로 해시 계산
- `printf "blob 5\0hello" | sha1sum`으로 Git 공식 직접 검증
- `git cat-file -p`로 parent 해시 체인 따라가기
- root commit(547a06c)까지 추적 — parent 없는 커밋 확인

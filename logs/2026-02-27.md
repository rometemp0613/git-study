# 2026-02-27 학습 일지

## 주제: submodule — 저장소 안의 저장소

### 배운 내용
- submodule: 내 저장소 안에 다른 저장소를 포인터(커밋 해시)로 포함
- 메인 저장소는 파일 내용이 아닌 커밋 해시만 기록 → 버전 고정
- submodule 폴더는 독립된 Git 저장소 (자기만의 remote, log, branch)
- 업데이트는 2단계: submodule에서 pull + 메인에서 커밋
- 삭제는 3단계: deinit → rm modules → git rm

### 실습
- /tmp에 원본 저장소(shared-lib)와 메인 프로젝트 생성
- submodule add로 추가 → .gitmodules 확인
- 원본에 새 커밋 후 메인에서 반영 안 됨 확인 (포인터 고정)
- submodule 안에서 pull → 메인에서 커밋으로 업데이트 완료
- submodule 삭제 3단계 실습

### 어려웠던 점
- submodule 안에서 git log를 치면 submodule 저장소의 로그가 나온다는 것 (독립된 저장소 개념)
- 로컬 파일 프로토콜 보안 문제 (`protocol.file.allow=always` 필요)

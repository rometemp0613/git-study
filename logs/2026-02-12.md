## 오늘 배운 것

### restore - 변경사항 취소

파일 단위로 변경사항을 되돌리는 명령어. reset/revert보다 간단하고 직관적.

### 핵심 명령어

| 명령어 | 용도 | 안전성 |
|--------|------|--------|
| `git restore <파일>` | 워킹 디렉토리 변경 취소 | 위험 (수정 내용 삭제) |
| `git restore --staged <파일>` | 스테이징 취소 (add 취소) | 안전 (내용 유지) |
| `git restore --source <커밋> <파일>` | 특정 커밋 시점으로 되돌리기 | - |
| `git restore .` | 모든 변경 파일 한 번에 되돌리기 | 위험 |

### 핵심 포인트

- **restore = 파일 단위**, reset/revert = 커밋 단위
- `--staged`는 안전 (내용은 워킹 디렉토리에 남음), `restore`만 쓰면 위험 (내용 날아감)
- staged + 워킹 모두 되돌리려면 `--staged` 먼저, 그다음 `restore`
- Git 2.23에서 checkout의 파일 복원 기능이 restore로 분리됨

### restore vs reset vs revert

| 명령어 | 대상 | 용도 |
|--------|------|------|
| restore | 파일 | 파일 변경사항/스테이징 취소 |
| reset | 커밋 | 커밋 히스토리를 되돌림 |
| revert | 커밋 | 취소 커밋을 새로 만듦 |

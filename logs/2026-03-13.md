# 2026-03-13 학습 일지

## 주제: 린트/포매팅 — 코드 품질 자동 검사

### 오늘 배운 것

1. **린트(Lint) vs 포매팅(Formatting) 차이**
   - 린트: 코드 품질 검사 (버그, 안 쓰는 변수, == 대신 === 등)
   - 포매팅: 코드 모양 통일 (들여쓰기, 따옴표, 세미콜론 등)

2. **ESLint 사용법**
   - `eslint.config.js` (Flat Config 방식)으로 규칙 설정
   - 규칙 레벨: off / warn / error
   - `globals` 패키지로 실행 환경(node/browser) 지정
   - `npx eslint src/` → `npx eslint --fix src/`

3. **Prettier 사용법**
   - `.prettierrc`로 스타일 설정 (semi, singleQuote, tabWidth 등)
   - `--check`: 검사만 (CI용)
   - `--write`: 실제 파일 수정
   - 옵션 없이 실행: 결과를 화면에만 출력

4. **ESLint + Prettier 함께 쓰기**
   - `eslint-config-prettier`로 충돌 방지
   - ESLint → 코드 품질, Prettier → 코드 모양 (역할 분담)

5. **GitHub Actions 연동**
   - `.github/workflows/lint.yml` 워크플로우 작성
   - PR마다 ESLint + Prettier 자동 검사
   - 일부러 에러 넣어서 실패 확인 → 수정 후 통과 확인

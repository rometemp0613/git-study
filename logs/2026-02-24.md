## 오늘 배운 것

### stash 기본
- `git stash`: 변경사항 임시 저장, 워킹 디렉토리 깨끗해짐
- `git stash pop`: 꺼내고 삭제 (= apply + drop)
- `git stash apply`: 꺼내고 목록에 유지
- `git stash list`: 목록 보기

### stash 옵션
- `-m "메시지"`: 설명 달기 (여러 개 쌓일 때 필수)
- `-u`: untracked 파일도 포함
- `stash@{n}`: 특정 stash 지정 (0이 가장 최근)

### stash 관리
- `git stash drop stash@{n}`: 특정 stash 삭제
- `git stash clear`: 전부 삭제

### 실습에서 발견한 것
- untracked 파일은 기본 stash에 안 들어감 → `-u` 필요
- stash는 브랜치에 종속되지 않음 → 다른 브랜치에서 pop 가능
- 이미 변경사항이 있는 파일에 stash apply하면 충돌 발생

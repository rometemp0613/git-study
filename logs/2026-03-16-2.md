## 학습 내용

### .git/ 폴더 핵심 구조
- **HEAD**: 현재 브랜치를 가리키는 텍스트 파일. `ref: refs/heads/main` 형태.
- **refs/heads/**: 로컬 브랜치. 브랜치 = 커밋 해시가 적힌 파일 1개.
- **refs/remotes/origin/**: 원격 추적 브랜치.
- **objects/**: blob, tree, commit 저장소. 해시 앞 2자리=폴더, 38자리=파일.
- **index**: Staging Area의 실체 (바이너리).
- **config**: 로컬 설정 (remote URL, branch 추적 등).
- **packed-refs**: refs를 한 파일로 합친 최적화.

### 핵심 발견
- `cat .git/refs/heads/main`과 `git log -1 --format="%H"` 결과 일치 → 브랜치는 해시 파일
- 파일 직접 생성으로 브랜치 만들기, rm으로 삭제 → 브랜치의 정체 직접 증명
- fetch 매핑 규칙: `+refs/heads/*:refs/remotes/origin/*`

## 실습
- `cat .git/HEAD`로 HEAD 파일 직접 확인
- `cat .git/refs/heads/main`과 `git log` 비교 (일치!)
- 파일 생성으로 브랜치 만들기 / rm으로 삭제
- `git ls-files --stage`로 Staging Area 내용 확인
- `cat .git/config`로 설정 파일 확인
- `cat .git/packed-refs`로 압축된 refs 확인

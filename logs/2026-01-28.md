## 오늘의 학습

### 진행한 내용
- Git 학습 시작
- 1단계 기초: 3가지 영역, 기본 명령어, amend
- 2단계 브랜치: branch, switch, merge

### 학습한 주제
- Git의 3가지 영역 (Working Directory, Staging Area, Repository)
- 기본 명령어 (init, add, commit, status, log)
- commit --amend
- 브랜치 기초 (branch, switch, merge)

### 새로 배운 명령어

| 명령어 | 설명 |
|--------|------|
| `git init` | 저장소 초기화 |
| `git add` | Staging Area로 파일 올리기 |
| `git commit` | 커밋 (스냅샷 저장) |
| `git status` | 현재 상태 확인 |
| `git log` | 커밋 히스토리 확인 |
| `git commit --amend` | 직전 커밋 수정 |
| `git branch` | 브랜치 관리 |
| `git switch` | 브랜치 이동 |
| `git merge` | 브랜치 합치기 |

### 메모 / 느낀 점
- Staging Area가 있어서 원하는 파일만 골라서 커밋 가능
- amend는 수정이 아니라 교체 (해시가 바뀜)
- push한 커밋은 amend 금지!
- `switch`가 `checkout`보다 브랜치 전용이라 더 명확함

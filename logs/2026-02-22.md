# 2026-02-22 학습 일지

## 주제: Issue & Project

### 배운 내용
- GitHub Issue: 할 일을 기록하는 티켓 (버그, 기능 요청 등)
- Issue 구성 요소: Title, Body, Assignee, Label, Milestone, Project
- Label: 이슈 분류 태그 (bug, enhancement, documentation 등)
- Milestone: 여러 이슈를 하나의 목표로 묶기
- Project: 칸반 보드로 이슈 시각적 관리 (Todo/In Progress/Done)
- `closes/fixes/resolves #번호`: PR 머지 시 이슈 자동 닫기

### 실습 내용
- `gh issue create`로 이슈 생성 (라벨 지정)
- `gh issue view`, `gh issue close`로 이슈 조회/닫기
- `gh issue edit --add-assignee @me`로 담당자 지정
- PR body에 `closes #11` 넣어서 머지 시 이슈 자동 닫기 확인

### 퀴즈 결과
- `fixes #번호`가 PR이 아니라 이슈를 닫는다는 점 혼동 → 정리 완료
- Issue 구성 요소: Milestone, Project를 빠뜨림 → 암기 필요
- `removes`는 자동 닫기 키워드가 아님 → 정답

### 메모
- `gh project list`는 `read:project` 스코프가 필요 (`gh auth refresh -s read:project`)
- PR 만들 때 반드시 커밋이 있어야 함 (브랜치만 push하면 "No commits" 에러)
